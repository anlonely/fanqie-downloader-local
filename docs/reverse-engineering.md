# 旧客户端下载链路与 Gateway 职责

## 背景

参考项目公开仓库：

- [POf-L/Fanqie-novel-Downloader](https://github.com/POf-L/Fanqie-novel-Downloader)

结合公开仓库说明、已发布客户端行为和本地提取结果，可以确认旧版客户端并不是直接在本地完成正文抓取和成书。

## 结论

旧版客户端主要负责：

- 本地 Web UI
- 本地任务状态展示
- 用户操作转发
- 从远端拉取最终产物

真正的核心工作由远端 gateway 完成，包括：

- 搜索
- 书籍详情预览
- 任务排队
- 章节抓取
- 正文解码
- TXT 或 EPUB 产物生成
- 最终文件分发

## Gateway 在做什么

从链路上看，gateway 至少承担这些职责：

1. 统一搜索和书籍详情接口
2. 接收下载任务并维护队列
3. 抓取章节正文
4. 处理番茄正文里的私有区字符
5. 生成可下载的成品文件
6. 提供产物下载接口给本地客户端拉取

## 这个本地版的替代方案

这个本地项目不再依赖远端任务网关，而是把关键流程直接放在本机：

1. 直接访问 `https://fanqienovel.com/page/<book_id>` 获取书籍信息和章节目录
2. 遇到章节链接时，先访问 `https://fanqienovel.com/reader/<item_id>` 反查所属书籍
3. 使用本地 `data/charset.json` 解码正文中的私有区字符
4. 在本地拼接全文并导出 TXT
5. 用 `data/session.json` 保存登录态，用 `data/jobs.json` 保存队列状态

## 当前本地版的边界

- 这是个人本地部署版，不依赖远端 gateway 服务
- 当前输出格式是 TXT
- 登录依赖用户自己浏览器中的 Cookie
- 仓库只保留本地可运行代码和必要文档，不包含历史提取产物
